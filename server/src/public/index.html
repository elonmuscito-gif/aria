The Black Box Problem

Companies are giving AI agents access to emails, payments, and databases. When an agent goes out of bounds, who is liable? You can't fix a compliance issue if you don't have a cryptographic proof of what the agent did.

How It Works

🔓

Cryptographic Identity

Every agent is registered with a unique DID and an HMAC-SHA256 signing key.

🛡️

Zero-Trust Scope

Declare what an agent can do (e.g., 'read:invoices'). ARIA enforces it server-side, ignoring client claims.

📊

Dynamic Reputation

Track success rates, errors, and anomalies. Know exactly which agents to trust.

Quickstart

TypeScript 
import { createClient } from '@elonmuscito/aria-sdk';

const aria = createClient({ 
  apiKey: 'key...' 
});

const agent = await aria.registerAgent({
  name: 'invoice-bot',
  scope: ['read:invoices']
});

await aria.track(agent.did, agent.secret, 'read:invoices', async () => {
  // Agent logic here
});

Distributed Trust Shell (DTS)

For high-stakes environments. ARIA splits the agent's secret using Shamir's Secret Sharing and binds it to a hardware fingerprint. The full secret never exists in your database or RAM.
